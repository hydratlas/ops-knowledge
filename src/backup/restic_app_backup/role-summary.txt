このロールは、ステートフルデータのバックアップにおけるアプリケーション層、すなわち各アプリが整合点をローカルで取得し restic リポジトリへ投入して世代を刈るまでを担うものである。アプリごとに1リポジトリ（/mnt/backup-dst/<job>、VIP NFS の rw マウント先）を持ち、各ノードは自分のリポジトリにしか書かないためノード間で restic ロックが競合しない。暗号鍵は全リポジトリ共通の単一鍵を Vault 変数 vault_restic_password で管理し、各ノードの /etc/restic/repo.pass へ配布する。restic は公式の静的バイナリ（restic_<version>_linux_amd64.bz2）を SHA256SUMS で検証して /usr/local/bin/restic へ配置し、OS 非依存の単一系統とする。ジョブは restic_app_backup_jobs（group_vars / host_vars から付与）のループで表現し、各ジョブの整合点取得を pre_command／post_command／source_path で差し込む。VictoriaMetrics・VictoriaLogs は snapshot API のハードリンク・スナップショットを安定パスへ mount --bind して restic の親照合（path,mtime,size,inode）を効かせ、新規 part のみを再ハッシュさせる。MariaDB は mariadb-dump --single-transaction、FreeIPA は ipa-backup（フル）で整合点を取る。整合点投入（日次・軽量な backup + forget）と pack 再パック（週次・重量な prune）を別スクリプト・別 systemd テンプレートタイマー（restic-backup@.timer / restic-prune@.timer）に分離し、重い prune の VIP NFS 越し I/O を日次経路から隔離する。両 service は flock で同一ロック（/run/restic-<job>.lock）を共有し同時実行を防ぐ。RandomizedDelaySec で複数ノードの I/O 集中を避ける。restic_app_backup_jobs が空のホストでは全タスクをスキップし、FreeIPA で idm-01 以外を no-op にする。
